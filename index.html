Enterprise AI runtime control

AI computers that can do useful work — safely.

ONEComputer gives teams an isolated workspace for Claude and other agents, enforces company policy at the network boundary, and routes high-impact actions through an external OpenVTC wallet.

The operating model

One business workflow. Two trust planes.

01

ONEComputer control plane

Administrators define company policy. Employees launch governed sandboxes. The gateway owns connector routing, enforcement, durable holds, and audit correlation.

+
02

OpenVTC trust plane

OpenVTC owns identity, trust authority, wallet key custody, versioned Trust Tasks, DIDComm/TSP delivery, and signed human approval evidence.

Admin policy→Employee sandbox→ Claude action→Gateway hold→ Wallet decision→One-time release

Security by construction

Keep the approval authority outside the app.

External wallet

Manager keys stay in an OpenVTC wallet, never in the web portal.

Fail closed

Missing, stale, altered, unauthorized, or replayed proofs do not release an action.

Least privilege

Credentials are injected by the gateway only for the governed connector and request.

Evidence first

Policy, hold, task, proof, and release share a durable correlation trail.

Open source boundary

Built as a thin layer over OpenVTC.

The product, trust infrastructure, mobile wallet, and protocol components are separate repositories so each can be upgraded and audited independently.
