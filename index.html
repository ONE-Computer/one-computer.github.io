The problem

AI can act. Your controls need to keep up.

Modern agents can browse, code, call APIs, and change real systems. A login screen is not a security boundary. ONEComputer gives each agent a real workspace while making every consequential action observable, policy-checked, and releasable only with the right proof.

Architecture

One business experience. Two independent trust planes.

ONEComputer owns the product workflow. OpenVTC owns identity, trust, and approval authority. The boundary is deliberate.

Security by construction

Approval authority stays out of the app.

The web UI can request and display status. It cannot impersonate the manager who approves a high-impact action.

01
External wallet

Signing keys remain in an OpenVTC wallet, isolated from the browser.

02
Fail closed

Stale, altered, unauthorized, or replayed proofs never release an action.

03
Evidence first

Policy, hold, task, proof, and release share one durable correlation trail.
